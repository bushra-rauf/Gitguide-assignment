Git Commands We Used to Build This Project throughout the development process, our team used several Git commands to collaborate and manage the codebase efficiently. Here are some of the essential commands we relied on:

git clone

Firstly we clone the project repository from Github to local machine

git branch

Create a new branch for feature development or bug fixes.

git checkout

Switch between branches to manage different features or stages of the project.

git merge

Merge feature branches back into the main branch once they have been reviewed and approved.

git pull origin

Pull the latest changes from a specific branch of the remote repository to ensure you are workingwith the most up-to-date code.

git add .

Stage all the changes made to the files, preparing them for commit.

git commit -m "commit message"

Commit the staged changes with a descriptive message that explains what was updated.

git push origin [branch-name]

Push the local commits to the remote repository, making them available for the team.
