Why Branching is Important

Some key reasons why branching is important:

Isolated Development:

Allows developers to work on different tasks without interfering with others.

Reduced Risk:

Changes can be tested and reviewed without affecting the main project.

Organized Workflow:

Teams can organize features, fixes, and releases more efficiently.

Versioning:

Preserves history, making it easy to trace or revert changes.

Git Commands for Branching

Branches are made from main

git checkout main

Creating a Branch

git checkout -b branch-name

Switching to a Branch

git checkout branch-name
