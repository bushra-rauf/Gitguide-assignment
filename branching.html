Why Branching is Important

Some key reasons why branching is important:

Isolated Development:

Allows developers to work on different tasks without interfering with others.

Reduced Risk:

Changes can be tested and reviewed without affecting the main project.

Organized Workflow:

Teams can organize features, fixes, and releases more efficiently.

Versioning:

Preserves history, making it easy to trace or revert changes.

Git Commands for Branching

Branches are made from main:

git checkout main

Creating a Branch

git checkout -b branch-name

Switching to a Branch

git checkout branch-name

Merging Branches

Why Merging is Important

Merging combines changes from different branches, allowing completed features or fixes to be integrated into the main branch or other branches. This process ensures that all team membersâ€™ contributions are combined into one cohesive project.

Steps for Merging a Branch

To merge a branch into the main branch, first switch to the main branch:

git checkout main

Then, use the following command to merge your feature branch into the main branch:

git merge branch-name

If Resolving Merge Conflicts

If there are conflicts (changes that cannot be automatically merged), Git will indicate the files that need attention. Open these files, review both changes, and edit them to resolve conflicts.

After resolving conflicts, mark the changes as resolved with:

git add .

Finally, complete the merge with:

git commit -m""
